# TXN-0 Automation Flows (Project Overview)

This repository contains the orchestration flows and documentation for **TXN-0**, an automation-native system built with **n8n + LLMs**.  
The system ingests user-submitted stories (via Google Form on [txn0.io](https://txn0.io)), validates them, and sends **symbolic on-chain acknowledgements**, while posting commentary to Telegram and X.

This repo is published as a technical reference ‚Äî not as a promotion ‚Äî to share how AI + workflow tools can be combined for real, auditable automation.

---

## üß† What It Does

- **Story Intake** ‚Üí Google Form (embedded on [txn0.io](https://txn0.io)) ‚ûú Google Sheets queue  
- **AI Validation** ‚Üí LLM scoring for relevance, duplication, and low-signal entries  
- **Wallet Handling** ‚Üí checksum + deduplication against `wallet-log.json`  
- **Airdrop Engine** ‚Üí sequential symbolic token sends with logging  
- **Ledger Logging** ‚Üí JSON file log as an off-chain mirror for auditability  
- **Cross-Posting** ‚Üí Telegram + X commentary with TX hash + truncated wallet details  
- **Ops** ‚Üí runs on Contabo VPS (Ubuntu) with cron-based backups and hardened `.env`

---

## üîí Safety & Guardrails

- **Sequential gate** ‚Üí processes one row at a time (prevents double-sends).  
- **Human-in-loop (Telegram)** ‚Üí optional approve/reject gate for edge cases.  
- **Social logic** ‚Üí strict filters; most branches return `SKIP` to avoid spam.  
- **Transparency** ‚Üí public X posts mirror BscScan TX hashes for every drop.

## üß© Dependencies

- **n8n** ‚Üí Node-based automation platform  
- **OpenAI API** ‚Üí GPT-4/mini for validation, scoring, and copywriting  
- **Telegram Bot API** ‚Üí approvals, alerts, and story moderation  
- **Google Sheets API** ‚Üí intake + queue (story submissions, tracking)  
- **Local JSON ledger** ‚Üí persistent file-based wallet log  
- **X (Twitter) API** ‚Üí replies, likes, RT, and announcement posts  
- **Contabo VPS (Ubuntu)** ‚Üí host with hardened `.env` and cron backups
